What is Hoos Sick?

We created Hoos Sick after being inspired by Apple phones and their ability to contact trace during the Covid-19 pandemic. We believe that this idea can be broadened to apply to any number of diseases, taking in location and additional data to compute risk for infection in response to how each illness is communicated.

Hoos Sick logs location data when users connect to the web app, which it then uses to construct tables of relevant location data that is parsed each time a user reports an infection. Our quad tree algorithm crunches the location and infection vectors of each reported illness to identify and notify those who were in communicative contact with the reporter.

Where do we go from here?

We would like to further this project by investigating bluetooth and similar networking protocols as methods for calculating cross-contact risk. Additionally, we plan on developing mobile applications that follow the Hoos Sick workflow so that more accurate/timely education data can be acquired in the background.

How do I use Hoos Sick?

User interaction drives data collection, which is then parsed by our algorithms to contact trace with as little user interaction as possible. Users can report infections as well as interactions, which prompts our app to analyze spread and notify individuals that we contact trace.

Hoos Sick is designed to be most effective when users actively update their location data by interacting with the web app. As location data is gathered, our algorithms are able to more effectively determine whether users have come into contact. As such, we see better contact tracing and thus reduced incidents of high-degree illness spread.
